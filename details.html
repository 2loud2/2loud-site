The details.

How the technology works

2Loud uses Google's YAMNet model — a pre-trained audio classification model that recognizes 527 sound categories. It runs locally in the browser via TensorFlow.js. No server, no cloud, no external processing.

The extension taps into the browser's Web Audio API — the same infrastructure that powers equalizers and bass boost extensions. It intercepts the audio stream from whatever you're watching, routes it through the model for classification, and adjusts the volume before it reaches your speakers.

A short buffer gives the model time to classify sounds before they play. When it detects a trigger — a doorbell, a gunshot, a scream — it smoothly lowers the volume and fades it back up after the sound passes.

The model doesn't listen through your microphone. It reads the audio data flowing from the video player. Your volume can be at zero and the model is still processing. It doesn't pick up your dog barking in the other room. It only processes what's coming from the browser tab.

It's just a filter

Audio comes out of the video player, passes through 2Loud, and continues to your speakers. Same way an equalizer works. Same plumbing, smarter logic.

The only difference is that instead of "cut everything below 200Hz," it's "cut everything the model thinks is a scream."

Development roadmap

Phase 1 — Proof of concept

  • Chrome extension that captures browser audio
  • YAMNet integration via TensorFlow.js for real-time classification
  • Basic volume lowering when a target sound is detected
  • Testing on YouTube first

Phase 2 — Core product

  • UI for selecting which sound categories to filter
  • Adjustable sensitivity and lowering intensity
  • Feedback mechanism (correct/incorrect classification)
  • Multi-platform testing (Netflix, Disney+, Hulu)
  • Chrome Web Store listing

Phase 3 — Dataset & community

  • Anonymous feedback aggregation pipeline
  • Public dashboard showing dataset growth and accuracy
  • Community-driven sound category tagging
  • Outreach to researchers and accessibility organizations
  • First public dataset release

Phase 4 — Expansion

  • Firefox extension
  • Model fine-tuning based on user feedback
  • Additional sound categories from community requests
  • Research partnerships
  • Academic paper on dataset findings

Session codes

An optional feature that bridges consumer product and research tool. A researcher, therapist, or educator can generate a session code — a short, temporary identifier tied to a study, not a person.

Participants enter the code in the extension. Their feedback flows into that study's bucket instead of the general pool. The researcher sees aggregate data — which sounds were flagged, confidence levels, misses and false alarms — without knowing anything about individual participants.

No accounts. No emails. No logins. Just a code that connects data points to a study.

The chain of insights

2Loud rests on a set of ideas that are individually obvious but collectively novel:

  1. The same AI used for security surveillance (sound classification) can be redirected at media consumption to help people rather than monitor them.
  2. A user feedback loop on AI accuracy naturally creates a dataset that nobody else can build.
  3. That dataset captures something science hasn't measured: real-time human sensitivity to classified sounds in media, at scale.
  4. All of this can be accomplished without knowing a single thing about the person using the tool.
  5. A non-profit structure makes every piece of this stronger, not weaker.
  6. The legal framework has existed since 2005, validated by two decades of ClearPlay's operation.

What 2Loud is not

  • Not an audio editor. It doesn't separate tracks. It lowers volume when trigger sounds are classified.
  • Not a censorship tool. It gives users control over their own audio environment.
  • Not a surveillance tool. It processes audio locally, stores nothing, transmits no audio.
  • Not a startup with an exit strategy. It's a non-profit built to serve a need and produce open data.
  • Not a finished product. As of this document, it's a working prototype being tested by friends.

Legal precedent

In 2005, Congress passed the Family Entertainment and Copyright Act (also called the Family Movie Act), which explicitly states that software enabling viewers to mute or skip parts of a movie does not violate copyright — as long as no fixed copy of the altered version is created.

This law was passed after Hollywood studios sued ClearPlay, a Chrome extension that filters profanity, violence, and nudity from streaming movies using human-annotated timestamps. ClearPlay won. It has operated legally for over two decades.

2Loud works the same way — modifying audio playback in real time, in the browser, for private home use, without creating copies. If anything, 2Loud is less legally complex than ClearPlay, because it classifies sound categories generically without knowing anything about the specific content being played.
